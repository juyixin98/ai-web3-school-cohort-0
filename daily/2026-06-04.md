## 关键收获

1. **Bridge AI Security 讲"是什么"，前沿版讲"怎么做"**：Bridge 章节讲 Prompt Injection / Tool Abuse / Key Safety 的概念和原则，前沿版直接给 Threat Model 画法、恶意文档 Demo 做法、工具权限 5 层划分法。两章互补——Bridge 建知识框架，前沿版给实施方案
2. **Agent Threat Model 的核心三问**：Agent 能读到什么、能调用什么、错了能不能追溯。风险按动作分层——只读链上数据最低、生成交易草稿中等、自动签名和授权最高。这直接对应 002 的 HITL 设计：高风险动作就是 Threat Model 里的"最高级"
3. **Tool Permission Isolation 的 5 层比 Bridge 版更实操**：公开只读 → 用户只读 → 草稿生成 → 写入工具 → 资产工具。越靠后越需要白名单/额度/模拟/确认/多签/延迟执行。005 的 Guard 在草稿层和资产层之间拦截
4. **Malicious Document Demo 是最小可行的安全入门练习**：正常文档里藏越权指令 → 观察系统是否标记外部资料、拒绝越权、不泄露 secret、调用前检查权限、输出拒绝原因。这个 Demo 很小但直接暴露最常见安全缺口——可以直接做成实验 006
5. **AI Behavior Audit 比 Bridge 的 Audit Log 更具体**：不是只记日志，而是确保出事后能定位到具体层——数据层/模型层/工具层/权限层/钱包层/界面层。每层有自己的审计粒度
6. **Privacy-preserving Workflow 连接了 AI Privacy 章节**：只读取必要数据、外部模型调用脱敏/摘要、默认隐藏不必要地址/身份、敏感任务优先本地模型/TEE
