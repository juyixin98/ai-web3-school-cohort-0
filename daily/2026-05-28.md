## 核心原则

> **所有进入模型的内容都可能是攻击面，所有离开模型的动作都必须被约束。**

## 关键概念

| 概念 | 一句话 | 难度 |
|---|---|---|
| **Prompt Injection** | 恶意内容藏在合约文档/网页/API返回中，试图覆盖 Agent 原始规则 | 中级 |
| **Tool Abuse** | 不是一次大动作，而是很多小动作累积——反复调用付费API、循环消耗预算 | 高级 |
| **Malicious Context** | 看似普通的上下文含攻击指令或假数据（假合约地址、伪造审计报告） | 中级 |
| **Key Safety** | 私钥/API key/session key 绝不能进入模型上下文或日志 | 高级 |
| **Permission Isolation** | 读工具和写工具分开、浏览器和钱包分开、普通任务和高风险任务分开 | 高级 |
| **Sandbox** | 隔离执行环境——默认禁止访问 .env/ssh key/wallet seed/生产凭证 | 中级 |
| **Audit Log** | 不只记「最终回答」，要记模型看了什么→为什么选这个工具→policy是否通过 | 初级 |
| **Alert** | 异常检测+响应：高频调用、重复付款、非白名单地址、参数偏离 → 暂停/撤销/冻结 | 中级 |
