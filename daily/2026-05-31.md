## 关键收获

1. **治理 AI 的核心是提高信息质量，不是替代政治判断**：AI 可以总结提案、提取会议 action、追踪贡献、审查预算，但投票和授权仍应由人或明确规则完成。治理比交易更重视正当性——AI 辅助工具必须透明、可质疑、可复核
2. **摘要的底线是来源和不确定性标注**：每个关键摘要都要能回到原始材料。事实（"提案请求 50,000 USDC"）和判断（"预算是否合理"）不能混在一起。无法给出来源的输出应明确标注为"推断"或"不确定"。治理场景宁可慢，不能用无来源摘要推动集体决策
3. **Contribution Graph 要能发现隐形贡献**：谁长期 review、谁协调会议、谁维护文档、谁回答新人问题——AI 可以整理证据，但不应把复杂贡献简化成 commits 数。与 Agent Trust 的连接：声誉信号需要贡献图提供原始数据，但同样要避免压成黑盒分数
4. **Budget Check 不是自动砍预算，而是让社区看清问题**：检查预算项完整性、里程碑具体性、历史拨款完成情况、金额与范围匹配度。链上维度还要检查收款地址归属、是否多签、是否需 vesting、历史有无异常提款。这是 Machine Payment 章节 Budget 概念在治理场景的延伸
5. **Plurality = 不要用平均立场抹平冲突**：AI 摘要不要只给"主流意见"，还要展示少数派担忧、利益相关者差异和可协商空间。一个费用提案，LP/交易者/开发者/金库管理者的视角不同——AI 应该展开这些视角，不是压成一个共识
6. **Meeting Action 的实质是把模糊讨论转成可检查事项**："我们应该研究一下"不是 action，"Alice 在下周五前整理三种支付方案"才是。决策状态要分层：已决定 / 待确认 / 需要投票 / 需要预算 / 需要法律复核
