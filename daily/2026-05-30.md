## 关键收获

1. **AI 主权的核心是四个能力：退出、迁移、选择、验证**。如果 Agent 管理钱包、治理投票、支付或私有记忆，用户需要的不只是"平台会保护你"，还需要可检查、可撤销、可迁移的机制。越靠近用户决策和资产的 AI，越不能只依赖平台承诺
2. **Data Portability 不是把原始数据打包下载**：要分层导出——任务历史 / 用户偏好 / 工具日志 / 身份凭证 / 私有记忆分别导出，标明敏感等级。给另一个 Agent 使用需要机器可读格式（JSON schema / VC / 签名日志），否则"可导出"只是下载一堆看不懂的文本
3. **Model Choice 不是让用户每次手动选模型，而是策略化选择**：隐私优先 → 本地模型、质量优先 → 强模型、成本优先 → 小模型。高风险 Agent 不能随便热切换模型——换模型会改变拒绝策略、工具调用风格和输出格式
4. **Local-first AI 连接了前两章**：本地做过滤/脱敏/风险初筛，云端强模型只收必要摘要，最终交易仍由钱包或智能账户确认。这是 AI Privacy 的 Local AI + AI Security 的 Guard 的融合方案
5. **CROPS 是一个产品价值检查表**：把 Ethereum 核心属性（抗审查、开源、隐私、安全）放到 AI×Web3 产品里逐项检验——Agent 身份是否依赖单平台、关键合约是否开源可检查、数据是否最小化隔离、钱包/权限/工具/日志是否有明确防线
6. **d/acc 一句话**：AI×Web3 不应该只做"更强的自动代理"，还要做"更好的防御工具"——更安全的钱包、抗钓鱼界面、可验证 Agent 日志、隐私保护数据协作、开源模型评测
