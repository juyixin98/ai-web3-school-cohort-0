## 关键收获

### Machine Payment 章节

1. **Budget 先于执行**：Agent 不应该拥有无限支付能力——先设边界（日预算/单笔上限/白名单/频率），再让 Agent 运行。常见错误是只设总预算不设频率和服务方范围
2. **Quote 是可追溯报价**：服务方返回的报价必须含价格/币种/有效期/quote_id，过期 quote 不能被继续使用，否则价格可能翻倍
3. **Payment Intent ≠ 已结算**：付款意图绑定任务+金额+收款方+有效期，给后续 audit 提供「这笔钱为什么花的」上下文

### Agent 受限支付钱包设计

4. **Guard 必须是确定性代码，不走 AI**：AI 推理可以被 Prompt Injection 或恶意上下文诱导，但代码规则不行。Policy 检查走确定性逻辑——金额超了就是超了，地址不在白名单就是不在
5. **三层决策**：小额自动（≤0.5 USDC）→ 大额 HITL（>0.5 但 ≤ 预算）→ 超预算直接拒（不给人判断，因为人在疲劳时也可能误点确认）
6. **撤销 > 授权**：用户随时能看到 Agent 花了多少、剩余多少、每一笔付给谁，并能一键关闭。系统也应在异常时自动撤销（到期/超预算/连续失败/异常地址探测）
