## 关键收获

### eth_call 实验

1. `eth_getBalance` 只能查原生 ETH，`eth_call` 可以调用任意合约的只读方法——这是 Agent 读取链上世界的主要窗口
2. Agent 调用合约需要三个东西：合约地址 + 函数选择器（keccak256 前 4 字节）+ ABI 编码参数
3. `eth_call` 不消耗 gas，不修改状态，Agent 可以放心用；危险的是 `eth_sendTransaction`

### Agent Wallet 章节

4. **核心原则**：控制权不能交给一个概率系统。Agent 只能拿到可验证、可限制、可撤销的行动空间
5. **Session Key** 是关键设计——不是把主私钥给 Agent，而是给一把临时的、有限额、限目标、限时间的「小钥匙」
6. **Policy + Guard 分工**：Policy 定义规则，Guard 是确定性拦截层。Agent 生成候选，Guard 负责拒绝

### 钱包交互地图

7. **6 步中只有 2 步 Agent 可完全自动**（读数据、查 Explorer），另外 4 步（连接、切换网络、签名、发交易）必须 HITL
8. **签名 ≠ 上链**——sign 不消耗 gas 且不修改链上状态，但它生成了「地址所有者同意内容」的密码学证明，所以仍然必须人工确认
9. **Agent 的核心价值不在执行**，而在执行前的 5 步准备：解析意图→查上下文→询价→生成交易→模拟风险——让人一眼看懂再决定
