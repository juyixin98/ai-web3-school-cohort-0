## 关键收获

### eth_call 实验

1. `eth_getBalance` 只能查原生 ETH，`eth_call` 可以调用任意合约的只读方法——这是 Agent 读取链上世界的主要窗口
2. Agent 调用合约需要三个东西：合约地址 + 函数选择器（keccak256 前 4 字节）+ ABI 编码参数。这是 Web3 Tool Use 的核心能力
3. `eth_call` 不消耗 gas，不修改状态，Agent 可以放心用；危险的是 `eth_sendTransaction`（需要签名+走 Workflow）

### Agent Wallet 章节

4. **核心原则**：控制权不能交给一个概率系统。Agent 只能拿到可验证、可限制、可撤销的行动空间
5. **Session Key** 是关键设计——不是把主私钥给 Agent，而是给一把临时的、有限额、限目标、限时间的「小钥匙」
6. **Policy + Guard 分工**：Policy 定义规则（每天最多花多少），Guard 是确定性拦截层（拒绝越界交易）。Agent 生成候选，Guard 负责拒绝

### 连接：eth_call → Workflow → Agent Wallet

```
eth_call (读)        → Step 2-5 全自动跑，安全
eth_sendTransaction  → Step 6 HITL 确认 + Guard 检查 + Session Key 权限边界
```
